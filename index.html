Project Abstract & Architecture

In higher education, academic curricula consist of complex prerequisite constraints. Manually auditing course plans is prone to human error and can inadvertently introduce unresolvable "cycles". This interactive system mathematically models courses as a Directed Graph using an Adjacency Matrix.

Core Objectives

  • Map real-world dependencies into a 2D computational grid.
  • Detect illegal circular dependencies using Depth-First Search (DFS) with 3-State Graph Coloring.
  • Generate pre-requisite and post-requisite chains using Breadth-First Search (BFS).

Algorithm Complexity

The system uses an Adjacency Matrix structure. While this allows blazing-fast $O(1)$ edge verifications, it scales at $O(V^2)$ for both Time and Space complexity, requiring careful management for extremely large course catalogs.

DFS Cycle Detection

A circular dependency (e.g., A requires B, and B requires A) breaks the curriculum. The system uses a recursive Depth-First Search with Graph Coloring to detect these loops. If the algorithm ever encounters a node currently marked as "Visiting", a cycle is mathematically confirmed.

BFS Path Traversal

Once the graph is cycle-free, we use Breadth-First Search (BFS) to isolate specific academic paths. Backward BFS scans matrix columns to find required ancestral courses (Prerequisites). Forward BFS scans matrix rows to find downstream unlocked courses (Post-requisites).
