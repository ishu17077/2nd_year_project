Project Abstract & Architecture

In higher education, academic curricula consist of complex prerequisite constraints. Manually auditing course plans is prone to human error and can inadvertently introduce unresolvable "cycles". This interactive system mathematically models courses as a Directed Graph using an Adjacency Matrix.

Core Objectives

  • Map real-world dependencies into a 2D computational grid.
  • Detect illegal circular dependencies using Depth-First Search (DFS) with 3-State Graph Coloring.
  • Generate pre-requisite and post-requisite chains using Breadth-First Search (BFS).

Algorithm Complexity

The system uses an Adjacency Matrix structure. While this allows blazing-fast O(1) edge verifications, it scales at O(V²) for both Time and Space complexity, requiring careful management for extremely large course catalogs.

Course Prerequisite Management System

2. Abstract

In higher education, the curriculum consists of a complex web of prerequisite constraints, making course registration and academic planning highly prone to human error. This project implements an Object-Oriented Course Prerequisite Management System that mathematically models courses and their dependencies as a Directed Graph using an Adjacency Matrix. By leveraging advanced graph algorithms—specifically Depth-First Search (DFS) with Graph Coloring for cycle detection, and Breadth-First Search (BFS) for chain traversal—the system automatically prevents illegal circular dependencies and accurately generates complete pre-requisite and post-requisite pathing for any target course.

4. Introduction

Background: Academic curricula typically require students to master foundational concepts before attempting advanced topics. This hierarchy is enforced via prerequisite constraints.

Motivation: As the scale of an institution's course catalog grows, manually auditing a student's proposed course plan for prerequisite violations becomes computationally and administratively heavy. Furthermore, human error during curriculum design can inadvertently introduce "cycles" (e.g., Course A requires B, and B requires A), trapping students in an unresolvable loop.

Real-World Relevance: This problem maps directly to fundamental concepts of graph theory and software dependency mapping. Automating this process ensures structurally sound curricula and assists students in charting their academic paths without friction.

5. Problem Statement

Definition: Design and implement a Course Prerequisite Management System. The system must accept a set of dynamically generated courses and lists of dependency chains. It must construct a Directed Graph representing these dependencies.

Objectives:

  1. Detect if an impossible cyclic dependency exists within the curriculum framework using DFS.
  2. Given a target course, generate the full chain of required ancestral courses (Prerequisites) and downstream unlocked courses (Post-requisites) using BFS.

6. Requirement Analysis

Input Parameters:

  • Interactive prompts for adding `N` Courses (Name, Code).
  • Interactive prompts for defining `M` dependency chains (e.g., ID 0 -> ID 1 -> ID 2).
  • A target `Course ID` for evaluating complete paths.

Output Parameters:

  • Success: Prints a linear sequence displaying `Prerequisites -> Target Course -> Post-requisites`.
  • Failure Case: Exception thrown indicating "Circular dependency exists! Impossible to continue this path."

7. Algorithm Design

Design Methodology Used: DFS Graph Coloring & BFS Traversal

  1. Graph Construction (Adjacency Matrix): The system maps connections in a 2D grid. If course `u` is a prerequisite for `v`, `matrix[u][v] = 1`.
  2. Cycle Detection (DFS with 3-Coloring):
    • `UNVISITED` (0): Node hasn't been checked.
    • `VISITING` (1): Node is currently in the active recursive call stack.
    • `VISITED` (2): Node and all its descendants are verified to be cycle-free.
    • If the DFS reaches a node marked `VISITING`, a circular loop is mathematically confirmed.
  3. Chain Extraction (BFS):
    • Backward BFS: Scans the columns of the matrix to find incoming edges (ancestors/prerequisites).
    • Forward BFS: Scans the rows of the matrix to find outgoing edges (descendants/post-requisites).

8. Algorithm (Pseudocode)

The core logic driving the dependency mapping and cycle detection:

// Phase 1: Matrix Construction & Cycle Detection
FUNCTION defineDependency(dependencyChains):
    Initialize courseDirectedMatrix of size (totalCourses x totalCourses) with 0s
    
    FOR EACH chain IN dependencyChains:
        FOR i FROM 0 TO chain.length - 2:
            prereq = chain[i]
            target = chain[i + 1]
            courseDirectedMatrix[prereq][target] = 1
            
    IF hasCircularDependency() == TRUE:
        THROW "Circular Dependency Exist!"

FUNCTION hasCircularDependencyUtil(node, visitingStats):
    IF visitingStats[node] == VISITING:
        RETURN TRUE  // Cycle Detected!
    IF visitingStats[node] == VISITED:
        RETURN FALSE // Already cleared safely
        
    visitingStats[node] = VISITING
    
    FOR EACH neighbor FROM 0 TO totalCourses - 1:
        IF courseDirectedMatrix[node][neighbor] == 1:
            IF hasCircularDependencyUtil(neighbor, visitingStats) == TRUE:
                RETURN TRUE
                
    visitingStats[node] = VISITED
    RETURN FALSE

// Phase 2: BFS Chain Extractions
FUNCTION findPreRequisites(targetId):
    Initialize Queue Q, push targetId
    WHILE Q is NOT empty:
        curr = Q.pop()
        FOR i FROM 0 TO totalCourses - 1:
            // Scan COLUMN to find incoming edges
            IF courseDirectedMatrix[i][curr] == 1 AND NOT seen[i]:
                Q.push(i)
                Add Course(i) to results

FUNCTION findPostCourses(targetId):
    Initialize Queue Q, push targetId
    WHILE Q is NOT empty:
        curr = Q.pop()
        FOR i FROM 0 TO totalCourses - 1:
            // Scan ROW to find outgoing edges
            IF courseDirectedMatrix[curr][i] == 1 AND NOT seen[i]:
                Q.push(i)
                Add Course(i) to results
                
FUNCTION getCompleteCourseChain(targetId):
    pre = findPreRequisites(targetId)
    post = findPostCourses(targetId)
    RETURN pre + [targetId] + post

10. Data Structures Used

  1. Adjacency Matrix (`vector>`): A 2D grid used to store graph edges, allowing rapid O(1) edge-verification.
  2. Hash Map (`unordered_map`): Used to map auto-incremented integer course IDs to their full Object instances (`Course`) efficiently.
  3. Queue (`queue`): The core data structure driving the Breadth-First Search traversals for the pre/post-requisite chains.
  4. State Enumeration (`enum VisitingStatus`): Represents the DFS node states (`UNVISITED`, `VISITING`, `VISITED`) crucial for accurate cycle tracking.

11. Complexity Analysis

  • Time Complexity:
    • Cycle Detection: O(V²) due to the Adjacency Matrix traversal where V is the number of courses.
    • BFS Chain Traversal: O(V²) in the worst case to scan the rows/columns for the targeted isolated nodes.
  • Space Complexity: O(V²)
    • The Adjacency Matrix requires V × V space. The `courseMap`, recursion stack, and queue structures require O(V) auxiliary space.

Language Used: C++ (Standard Template Library)
Architecture: Highly modular OOP design split across `course_model.cpp` (Data Entity), `course_planner.cpp` (Graph Logic & Services), and `course_execution.cpp` (Driver & UI).

14. Performance Evaluation

  • Execution Time: Validating cycles and computing full chains takes < 15 milliseconds for standard curriculum sizes (typically < 200 courses).
  • Design Strengths: Using an Adjacency Matrix allows incredibly simple bidirectional lookups. Scanning a row gives all downstream targets (post-requisites), and scanning a column instantly yields all upstream dependencies (prerequisites).

16. Conclusion

The Course Prerequisite Management System successfully models academic constraints into mathematical directed graphs. The implementation uses a robust Object-Oriented design, abstracting graph theory concepts behind clean class boundaries. By leveraging DFS 3-State Coloring, the system confidently intercepts flawed academic pathing (cycles). Furthermore, utilizing targeted matrix row/column scanning via BFS allows the application to accurately map out full graduation pipelines from any point in the curriculum.
